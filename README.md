# U_TTS: Мультидвижковый TTS-сервер для Twitch

**U_TTS**beta — это комплексное программное решение для синтеза речи (TTS), предназначенное для озвучивания сообщений чата и событий Twitch (подписки, донаты, рейды, Hype Train и др.). Приложение функционирует как резидентная программа в системном трее Windows и предоставляет интуитивно понятный веб-интерфейс для управления, мониторинга и тонкой настройки.

## 🚀 Основные возможности

*   **Мультидвижковая архитектура**: Поддержка 6 различных TTS-движков (облачные, локальные CPU, нейросетевые GPU).
*   **Клонирование голоса**: Возможность загрузки референсных `.wav` файлов для XTTSv2, Qwen3-TTS и CosyVoice3.
*   **Глубокая интеграция с Twitch**:
    *   Подключение к чату по нику канала.
    *   Поддержка **EventSub** (OAuth 2.0) для отслеживания подписок, донатов (Bits), рейдов, челленджей и целей канала.
*   **Персонализация**: Привязка уникального голоса, движка и параметров (скорость, тон) к конкретным зрителям с хранением в базе данных SQLite.
*   **Гибкая система фильтрации**: Блокировка ссылок, эмоутов, emoji; настройка черных/белых списков, кулдаунов и триггерных ключевых слов.
*   **Современный веб-интерфейс**: Темная тема (Bootstrap 5), логи в реальном времени (SSE), поддержка эмоутов (Twitch, BTTV, 7TV, FFZ) и тестовое воспроизведение.
*   **Управление из трея**: Запуск/остановка сервера и воркеров, автоматическая очистка занятых портов.
*   **Монолитная сборка**: Возможность компиляции в единый `.exe` файл с помощью PyInstaller.

## 🎙 Поддерживаемые TTS-движки

| Движок | Тип | Требуется GPU | VRAM | Описание и доступные голоса |
| :--- | :--- | :---: | :---: | :--- |
| **edge-tts** | Облачный (Microsoft Edge) | Нет | — | 400+ голосов (все языки), 2 русских голоса. Идеально для быстрого старта. |
| **SAPI5** | Локальный (Windows) | Нет | — | Системные голоса Windows. |
| **Piper** | Локальный (ONNX, CPU) | Нет | — | 3 русских голоса в комплекте. Высокая скорость работы на CPU. |
| **XTTSv2** | Docker / Процесс | Да (CUDA) | ~4 ГБ | Клонирование голоса + встроенные голоса. Оптимален для слабых GPU. |
| **Qwen3-TTS** | Docker / Процесс | Да (CUDA) | ~6 ГБ | Продвинутое клонирование голоса (модель `Qwen3-TTS-12Hz-0.6B`). |
| **CosyVoice3** | Docker / Процесс | Да (CUDA) | ~6 ГБ | Высококачественное клонирование голоса (модель `Fun-CosyVoice3-0.5B`). |

### 💡 Рекомендации по выбору
*   **Для быстрого старта (без GPU):** `edge-tts`, `SAPI5`, `Piper`.
*   **Для клонирования голоса на弱ом GPU:** `XTTSv2` (~4 ГБ VRAM).
*   **Для максимального качества клонирования:** `Qwen3-TTS` или `CosyVoice3` (~6 ГБ VRAM).

## 💻 Системные требования

### Базовые (для edge-tts, SAPI5, Piper)
*   **ОС:** Windows 10+ (поддержка трея, named mutex)
*   **Python:** 3.10+
*   **Диск:** ~500 МБ (ядро + Piper + edge-tts)
*   **Сеть:** Доступ в интернет (Twitch API + edge-tts)

### Расширенные (для XTTSv2, Qwen3-TTS, CosyVoice3)
*   **GPU:** NVIDIA с поддержкой CUDA 12.4+
*   **VRAM:** 8–16 ГБ (рекомендуется)
*   **Диск:** 10–30 ГБ (модели + зависимости)
*   **Среда:** Docker Desktop (WSL2 + виртуализация) и NVIDIA Container Toolkit.

## ⚙️ Установка и настройка

### 1. Подготовка окружения
Запустите скрипт для создания виртуального окружения и установки зависимостей:
```bat
install_myenv.bat
```

### 2. Инициализация конфигурации
Скопируйте шаблоны конфигурационных файлов в рабочую директорию `data/`:
```bat
copy config.example.json data\config.json
copy twitch_oauth.example.json data\twitch_oauth.json
```

### 3. Настройка Twitch OAuth
1. Перейдите в [Twitch Developer Console](https://dev.twitch.tv/console).
2. Создайте приложение, указав **OAuth Redirect URL**: `http://localhost:3000`.
3. Скопируйте `Client ID` и `Client Secret` в файл `data/twitch_oauth.json`:
   ```json
   {
     "client_id": "ваш_client_id",
     "client_secret": "ваш_client_secret"
   }
   ```

### 4. Запуск приложения
Запустите резидентное приложение:
```bat
tray_app.bat
```
*Или напрямую через интерпретатор:* `myenv\Scripts\python.exe tray_app.py`

> **Примечание:** При первом запуске автоматически откроется браузер для прохождения авторизации Twitch. Веб-интерфейс будет доступен по адресу: [http://127.0.0.1:5000](http://127.0.0.1:5000).

### 5. Загрузка моделей Piper (Опционально)
Модели Piper необходимо скачать вручную из-за их размера.
*   **Голоса (ONNX + JSON):** Поместите в `data/voices/piper/` ([Скачать с HuggingFace](https://huggingface.co/rhasspy/piper-voices/tree/main/)).
*   **Бинарные файлы воркера:** Поместите в `workers/piper_tts/` ([Скачать Piper Windows AMD64](https://github.com/rhasspy/piper/releases/tag/2023.11.14-2)).

## 🐳 Управление Docker-воркерами

Для нейросетевых движков требуется запущенный Docker. Перейдите в соответствующую директорию и разверните контейнер:

**Qwen3-TTS** (Порт: 5002)
```bash
cd workers\qwen3_tts
docker compose up -d
```

**XTTSv2** (Порт: 5004)
```bash
cd workers\xtts
docker compose up -d
```

**CosyVoice3** (Порт: 5003)
```bash
cd workers\cosyvoice
docker compose up -d
```

*Для одновременного запуска всех воркеров используйте корневой `docker-compose.yml`:*
```bash
docker compose up -d
```

## 🌐 Веб-интерфейс и API

Веб-интерфейс предоставляет полный контроль над системой, включая настройку фильтров, управление событиями и тестирование голосов. Обмен данными в реальном времени осуществляется через Server-Sent Events (SSE).

### Основные эндпоинты API

| Маршрут | Метод | Описание |
| :--- | :--- | :--- |
| `/` | GET | Главная страница веб-интерфейса |
| `/api/status` | GET | Текущий статус сервера и воркеров |
| `/api/config` | GET/POST | Чтение и обновление `config.json` |
| `/api/generate` | POST | Генерация TTS (параметры: text, engine, voice) |
| `/api/tts/stream` | GET | Потоковое воспроизведение аудио |
| `/api/voices` | GET | Список доступных голосов для выбранного движка |
| `/api/voices/upload` | POST | Загрузка референсного `.wav` для клонирования |
| `/api/logs` | GET | Получение последних 50 записей лога |
| `/api/sse` | GET | Подключение к потоку Server-Sent Events |
| `/api/emotes` | GET | Загрузка кэша эмоутов (BTTV, 7TV, FFZ, Twitch) |
| `/api/twitch/start` | POST | Инициализация подключения к чату Twitch |
| `/api/twitch/stop` | POST | Отключение от чата Twitch |
| `/api/twitch/channel`| POST | Смена целевого канала |
| `/api/send_chat` | POST | Отправка текстового сообщения в чат от имени бота |

## 📂 Структура проекта

```text
├── audio_player/            # Аудиоплеер (sounddevice, Flask, порт 5006)
├── core/                    # Ядро приложения
│   ├── templates/           # Frontend (HTML/CSS/JS)
│   ├── _edge_helper.py      # Асинхронный helper для edge-tts
│   ├── addon_manager.py     # Оркестрация воркерами (Docker/Процессы)
│   ├── config_store.py      # Thread-safe работа с JSON-конфигом
│   ├── homographs.py        # Словарь омографов (рус. язык)
│   ├── piper_engine.py      # Интеграция Piper TTS (ONNX)
│   ├── runtime_events.py    # Система логирования и SSE
│   ├── sentence_boundary.py # Алгоритмы разбивки текста на предложения
│   ├── tts_engine.py        # Базовые движки (edge-tts, SAPI5)
│   ├── twitch_auth.py       # OAuth 2.0 авторизация
│   ├── twitch_eventsub_api.py # Работа с EventSub
│   ├── voice_map_db.py      # SQLite: маппинг пользователей и голосов
│   └── yo.txt               # Словарь ё-фикации (~58k слов)
├── data/                    # Пользовательские данные (игнорируется в Git)
│   ├── config.json          # ★ Основной конфиг
│   ├── twitch_oauth.json    # ★ Twitch credentials
│   ├── voices/              # Референсные аудио и модели Piper
│   ├── outputs/             # Кэш сгенерированных аудиофайлов
│   └── latents/             # Кэш эмбеддингов XTTS
├── workers/                 # Изолированные TTS-воркеры
│   ├── cosyvoice/           # CosyVoice3 (Docker)
│   ├── piper_tts/           # Piper (Локальные бинарники)
│   ├── qwen3_tts/           # Qwen3-TTS (Docker)
│   └── xtts/                # XTTSv2 (Docker)
├── server.py                # Flask-сервер (порт 5000)
├── irc_bot.py               # Клиент Twitch IRC
├── tray_app.py              # Трей-приложение (pystray)
├── docker-compose.yml       # Root Docker Compose
└── requirements.txt         # Зависимости Python
```

## 🛠 Дополнительные утилиты

*   **Сборка в EXE:** Запустите `build_exe.bat` для компиляции проекта в автономный файл `dist/U_TTS.exe` с помощью PyInstaller.
*   **Очистка портов:** Запустите `cleanup.bat` для принудительного завершения процессов, зависших на портах `5000, 5003–5006, 3000`, а также всех фоновых процессов `python.exe`, связанных с U_TTS.
*   **Логирование:** Системные логи дублируются в файл `data/server.log`.
*   **Пользовательские словари:** Файлы `core/user.txt` (ударения) и `core/yo.txt` (ё-фикация) позволяют корректировать произношение.
